1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout (. - если все файлы, имя файла - если конкретный файл)

2. Как вернуть файл из индекса в рабочий каталог?
git restore --stagef <file>

3. Как отменить ошибочный commit ?
 git reset HEAD~ (откатить к предыдущему комиту)
 git reset HEAD^ (вернуться к одному из предыдущих комитов, где ^ - количество шагов назад)
 git revert HEAD <код комита> + git push 

4. Как создать новую ветку и переключится в нее?
git checkout <название ветки> - создать ветку или переключиться на ветку, если такая уже есть
git checkout -b <название ветки> - создать ветку и переключиться на нее

5. В чем отличение между fast-forward merge  and non fast-forfard merge?
fast - это простое слияние веток (когда нет конфликтов)
non fast - когда есть конфликт слияния и git требует сравнить файлы и указать, какой вариант корректен

6. Когда может возникнуть конфликт слияния веток, как  его разрешить?
Конфликт возникает, когда в разных ветках в одном и том же файле (файлах) разные комиты.
Нужно вручную отредактировать до корректной версии, потом git add и git commit

7. Можно ли удалить ветку? Как? 
Можно. git branch -D <название ветки>
